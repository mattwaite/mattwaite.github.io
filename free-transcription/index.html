Audio Transcription Guide

Transcribing Audio Files on Your Own Computer

A step-by-step setup guide using uv and Faster Whisper — no prior technical experience needed.

What's in this guide

  1. Open your Terminal (Mac) or Command Prompt (Windows)
  2. Install uv — a Python manager
  3. Create a project folder and Python environment
  4. Install Faster Whisper
  5. Transcribe your first audio file
  6. Saving your transcript to a file
  7. Common options and tips
  8. Troubleshooting

This guide walks you through installing everything you need to transcribe audio files on your own laptop — no internet connection required once it's set up, and no files are sent to any outside service. Everything runs locally on your machine.

We'll use a tool called uv to handle Python installation for us, which avoids most of the confusing issues people run into with Python on Macs and Windows.

⏱ Time estimate: This setup takes about 10–15 minutes the first time. After that, transcribing a file takes just one command.
1
Open your Terminal or Command Prompt

All the commands in this guide are typed into a text interface called the Terminal (Mac) or Command Prompt (Windows). Think of it as a way to give your computer very direct instructions by typing them.

# Press Command (⌘) + Spacebar to open Spotlight Search
# Type: Terminal
# Press Enter

# A white or black window with a blinking cursor will appear.
# That's your Terminal. You're ready.
# Press the Windows key, then type: cmd
# Click "Command Prompt" in the results

# A black window with a blinking cursor will appear.
# That's your Command Prompt. You're ready.
💡 When this guide says "type a command," it means: click inside the Terminal/Command Prompt window, type exactly what's shown, and press Enter. The computer won't do anything until you press Enter.
2
Install uv

uv is a modern tool that installs Python and manages your Python projects. It avoids the confusing version conflicts that come with the default Python on Macs and Windows machines.

Copy and paste the command below for your operating system, then press Enter:

curl -LsSf https://astral.sh/uv/install.sh | sh
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

You'll see a bunch of text scroll by as it downloads and installs. This is normal. When it finishes, you'll see your cursor again.

Mac users — one extra step: After uv installs, it may tell you to run one more command to "update your shell profile." If you see a line starting with source, copy and paste that line and press Enter too. Then close your Terminal window and open a fresh one before continuing.

Verify the installation worked

Type this command and press Enter:

uv --version

You should see something like uv 0.5.x. If you see that, uv is installed and working.

3
Create a project folder and Python environment

We'll create a dedicated folder for your transcription work, then set up an isolated Python environment inside it. Think of this like a clean workspace — any software installed here stays separate from the rest of your computer.

# Create a folder called "transcription" on your Desktop
mkdir ~/Desktop/transcription

# Move into that folder
cd ~/Desktop/transcription

# Create a Python 3.12 environment (must be 3.12, not 3.13)
uv venv --python 3.12

# Activate the environment
source .venv/bin/activate
:: Create a folder called "transcription" on your Desktop
mkdir %USERPROFILE%\Desktop\transcription

:: Move into that folder
cd %USERPROFILE%\Desktop\transcription

:: Create a Python 3.12 environment (must be 3.12, not 3.13)
uv venv --python 3.12

:: Activate the environment
.venv\Scripts\activate
✓ You'll know the environment is active when your command prompt changes to show (.venv) at the beginning of the line, like: (.venv) your-computer:transcription $
Why Python 3.12 specifically? Faster Whisper depends on a library called ctranslate2 that doesn't yet support Python 3.13 (the newest version). Using 3.12 avoids a frustrating error.
Important: Every time you open a new Terminal window in the future to do transcription, you'll need to navigate back to your folder and re-activate the environment. The two commands are:

Mac cd ~/Desktop/transcription then source .venv/bin/activate
Win cd %USERPROFILE%\Desktop\transcription then .venv\Scripts\activate
4
Install Faster Whisper

With your environment active (you should see (.venv) in your prompt), run this command. It works the same on both Mac and Windows:

uv pip install faster-whisper-cli

This will download Faster Whisper and everything it needs. The first time will take a minute or two depending on your internet speed. You'll see a progress bar as packages download.

Verify the installation

faster-whisper --help

If you see a list of options and flags, you're all set. Faster Whisper is installed and ready to use.

5
Transcribe your first audio file

Copy your audio file (MP3, WAV, M4A, or most other formats) into your transcription folder on your Desktop. Then run:

# Replace "myaudio.mp3" with the actual name of your file
faster-whisper myaudio.mp3
First run takes longer. The first time you run Faster Whisper, it downloads an AI model (about 150MB for the default "small" model). This happens automatically and only happens once. After that, transcription is fast.

What you'll see

EXAMPLE OUTPUT
[00:00.000 --> 00:03.240] Welcome to today's interview. My name is Sarah.
[00:03.240 --> 00:07.810] We're going to talk about the proposed changes to the zoning ordinance.
[00:07.810 --> 00:12.440] Joining me is council member James Rivera from District 4.
...

Each line shows a timestamp followed by the transcribed text. The transcript prints directly to your Terminal window.

Accuracy note: Faster Whisper is excellent but not perfect. Proper nouns, technical jargon, and heavily accented speech may need manual correction. Always review your transcript before using it.
6
Saving your transcript to a file

By default, the transcript just prints to the screen. To save it to a text file you can open in any text editor, use the -o flag followed by the output filename:

# Save transcript to a file called "transcript.txt"
faster-whisper myaudio.mp3 -o transcript.txt

The file will appear in your transcription folder on your Desktop. You can open it in any text editor (TextEdit on Mac, Notepad on Windows, or Word).

Transcribing multiple files

To transcribe several files, run the command once for each file, giving each a different output name:

faster-whisper interview1.mp3 -o interview1_transcript.txt
faster-whisper interview2.mp3 -o interview2_transcript.txt
faster-whisper meeting.m4a -o meeting_transcript.txt
7
Useful options

You can add flags to the command to change how the transcription works. Here are the most useful ones:

FlagWhat it doesExample
-o FILE Save output to a text file instead of printing to screen -o transcript.txt
--language Tell Whisper what language is spoken (speeds things up; auto-detects by default) --language en
--model Choose a larger model for better accuracy (uses more memory and time) --model medium
--task translate Translate non-English audio to English text --task translate
--word_timestamps True Include timestamps for every word, not just each segment --word_timestamps True

Model sizes

Faster Whisper comes with several model sizes. Bigger = more accurate but slower:

ModelSpeedAccuracyGood for
tinyVery fastBasicQuick drafts, clear speech
baseFastGoodMost everyday use
smallModerateVery goodDefault — good balance
mediumSlowExcellentInterviews, noisy audio
large-v3Very slowBestWhen accuracy is critical

Example using medium model with English language specified:

faster-whisper interview.mp3 --model medium --language en -o transcript.txt
8
Troubleshooting

"faster-whisper: command not found"

Your virtual environment is not active. Navigate to your transcription folder and activate it:

cd ~/Desktop/transcription
source .venv/bin/activate
cd %USERPROFILE%\Desktop\transcription
.venv\Scripts\activate

"No such file or directory" when running faster-whisper

The audio file isn't in your transcription folder, or the filename is spelled differently. Make sure your file is in ~/Desktop/transcription (Mac) or %USERPROFILE%\Desktop\transcription (Windows). Filenames are case-sensitive on Mac.

Transcription seems to hang or is very slow

This is normal for larger audio files and larger models. A one-hour recording on the medium model can take 10–15 minutes on a laptop CPU. Try the small or base model for faster results, or let it run and be patient.

"uv: command not found" after restarting Terminal

On Mac, close your Terminal and open it fresh. If uv is still not found, run the install command from Step 2 again.
